# 需求文档 — MC Mod Hub 桌面版

## 产品名称

**MC Mod Hub 桌面版**

## 一句话介绍

一个 Windows 桌面软件，帮你集中搜索、管理、批量下载 Minecraft Mod/光影/材质包，装上就能用，无需租服务器。

## 目标用户

- 我自己，以及我小圈子里的朋友们（国内 Minecraft 玩家）
- 都是 Windows 用户
- 不需要注册登录，装上即用

## 使用场景

我有好几个不同的 MC 游戏档（比如 1.21 生存档、服务器联机档、1.20 养老档），每个档装的 Mod 不一样。每次换档或换版本，都得去 CurseForge、Modrinth 上重新搜 Mod、找版本、一个个下载。

有了这个软件，我把所有喜欢的 Mod 按游戏版本分类存进收藏夹，什么时候要用，打开软件勾选几个就能批量下载。想直接下一个 Mod 也行，搜到就能马上下。

## 要解决的核心问题

1. 网页版每次要用都得手动启动服务器，太麻烦
2. 网页版想分享给朋友用，还得教他们怎么装 Node.js、npm install、npm run dev，门槛太高
3. 不想花钱租服务器部署
4. Mod 分散在 CurseForge 和 Modrinth 两个平台，搜索和管理不统一
5. 不同游戏版本需要的 Mod 不同，缺少统一管理工具

## 第一版功能范围

以下四个功能**全部必须有**，缺一不可：

### 搜索
- 输入关键词，同时搜索 CurseForge 和 Modrinth 两个平台
- 搜索结果合并展示，去重
- 可以按 Mod / 光影 / 材质包筛选

### 收藏夹管理
- 创建/删除/重命名收藏夹
- 每个收藏夹绑定一个游戏版本（如 1.21、1.20.1）
- 从搜索结果或热门列表添加资源到收藏夹
- 从收藏夹中移除资源
- 收藏夹内可筛选：按加载器（Forge/Fabric/Quilt）、按发布类型（正式版/Beta/Alpha）、按游戏版本

### 资源详情与版本选择
- 查看资源基本信息（名称、图标、描述、作者、下载量）
- 查看资源的所有可用版本列表
- 为收藏夹中的每个资源独立选择下载版本
- 支持直接下载单个 Mod（不进收藏夹）

### 批量下载
- 勾选收藏夹中的多个资源
- 选择目标游戏版本，自动匹配对应版本文件
- 两种下载方式由用户自由选择：
  - **打包成 zip**：所有文件下载完成后自动压缩成一个 zip，弹出保存对话框
  - **逐个下载到文件夹**：所有文件下载到用户指定的文件夹
- 下载失败的文件单独提示，不影响其他文件

## 第一版不做什么（以后再说）

- 不增加新功能，照搬网页版已有功能
- 不做用户登录/注册
- 不做收藏夹分享
- 不做兼容性检查（网页版已有，但第一版先砍掉，降低复杂度）
- 不做手机端
- 不做深色/浅色主题切换（只做深色 Minecraft 风格）
- 不做国际化（全中文）

## 页面列表

| 页面 | 路由 | 作用 |
|------|------|------|
| 首页 | `/` | 搜索栏 + 热门 Mod/光影/材质包板块 + 最近浏览 + 更新提醒 |
| 资源详情页 | `/resource/:source/:id` | 查看资源详情、选版本、直接下载 |
| 收藏夹列表页 | `/collections` | 查看所有收藏夹、新建/删除/重命名收藏夹 |
| 收藏夹详情页 | `/collections/:id` | 资源列表 + 版本选择 + 筛选 + 批量下载 |
| 设置页 | `/settings` | 填写 CurseForge API Key、选择下载目录等 |
| 分类浏览页 | `/category/:type` | 按 Mod/光影/材质包分类浏览全部资源 |
| 更新提醒页 | `/updates` | 收藏夹资源更新列表 |

## 用户操作流程

### 流程 1：直接搜索并下载
1. 打开软件 → 首页搜索栏输入 Mod 名称
2. 从搜索结果找到想要的 Mod → 点击进入详情页
3. 选择需要的版本 → 点"下载"按钮
4. 文件直接下载到电脑

### 流程 2：管理收藏夹并批量下载
1. 在搜索结果或热门列表中找到想要的 Mod
2. 点击快捷菜单 → "添加到收藏夹" → 选择目标收藏夹
3. 打开收藏夹详情页 → 为每个 Mod 选择需要的版本
4. 勾选要下载的 Mod → 选择下载方式（zip 打包 / 逐个下载）→ 点下载
5. 系统自动下载并保存到用户指定位置

### 流程 3：首次使用设置
1. 打开软件 → 进入设置页
2. 填入自己申请的 CurseForge API Key
3. 选择默认下载目录
4. 回到首页开始使用

## 最终交付效果

一个 Windows 桌面软件：
- 双击就能打开，不需要装 Node.js，不需要跑命令行
- 首页搜索 Mod、浏览热门内容
- 资源详情页选版本、直接下载
- 收藏夹管理自己的 Mod 库
- 一批批勾选、批量下载
- 所有数据保存在本地电脑上
- 软件安装包发布到 GitHub，朋友下载安装即可

## 开发步骤索引

整个开发分为 **10 个独立步骤**，每步在一个新对话中完成。每步有明确的输入（上一步产出）和输出（本步交付物）。

| 步骤 | 名称 | 一句话目标 | 依赖 |
|------|------|-----------|------|
| Step 1 | 项目初始化 | 配好所有配置文件，`npm run dev` 能启动 | 无 |
| Step 2 | 基础设施层 | 类型 + 数据库 + API 客户端全部就绪 | Step 1 |
| Step 3 | 全局 UI + 导航 | 看到 Minecraft 风格导航栏和页面框架 | Step 2 |
| Step 4 | 首页 — 搜索 + 热门 | 首页完整功能：搜索栏 + 三大热门板块 + 资源卡片 | Step 3 |
| Step 5 | 资源详情 + 单文件下载 | 卡片→详情页→选版本→下载成功 | Step 4 |
| Step 6 | 收藏夹管理 | 收藏夹 CRUD + 快捷菜单添加资源 | Step 5 |
| Step 7 | 收藏夹详情 + 批量下载 | 版本选择 + 筛选 + zip/文件夹批量下载 | Step 6 |
| Step 8 | 补充功能 | 最近浏览 + 更新提醒 + 分类浏览 | Step 7 |
| Step 9 | 设置页 + Electron 壳 | Electron 窗口 + API Key 设置 + 原生对话框 | Step 8 |
| Step 10 | 打包 + GitHub 发布 | exe 安装包发布到 GitHub Release | Step 9 |

**使用方式**：每步开一个新对话，把对应步骤的说明（见 `technical-design.md` 第 17 节）告诉 AI 即可。

## 应急功能协议

当需要在开发过程中添加一个新功能时，按以下流程操作，确保不影响已有步骤：

### 操作流程

1. **确定插入位置**：新功能最自然属于哪个步骤之后？把它插在那个步骤和下一步骤之间（如 Step 6.5）
2. **创建功能说明**：在 `technical-design.md` 的步骤索引中新增一条
3. **开新对话开发**：用以下模板告诉 AI：
   > 我需要在 Step X 完成后插入一个新功能：[功能描述]。请参考 `requirements.md` 和 `technical-design.md`，仅创建新文件或扩展已有文件。禁止修改已有功能的核心逻辑。

### 安全规则（保护已有功能）

- ✅ **可以做的**：新建组件文件、新建 API 路由、新建页面、在 `src/types/index.ts` 末尾新增接口字段、在 `electron/preload.ts` 末尾新增 IPC 方法、在导航栏末尾新增链接
- ❌ **禁止做的**：修改已有数据库表结构（只允许新增表）、修改已有页面核心逻辑、修改已有 API 接口签名、修改 Tailwind 色板已有颜色值、删除任何已有代码或文件

### 新功能开发条件检查清单

开发前确认以下三点即可开始：
- 新功能不修改已有数据库表的列（可以新增表）
- 新功能不改变已有 API 的请求/响应格式（可以新增 API 路由）
- 新功能的 UI 使用已有组件库中的样式 token（不从零定义颜色）
